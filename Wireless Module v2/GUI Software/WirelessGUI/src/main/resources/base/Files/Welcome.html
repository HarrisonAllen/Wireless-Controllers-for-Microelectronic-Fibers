Welcome!

A quick note on the GUI: every window/tab can be rearranged. You can move each window around, dock them side-by-side, or combine windows into a tab group. Try it out!

Getting Started

  1. Plug in the Central board to your computer.
  2. Switch to the Central tab(a).
  3. Select the corresponding port from the dropdown(b).
    • In this example, it is COM6. It will likely be something different on your machine.
    • If you do not see the correct port in the dropdown, then try pressing Refresh(c).
  4. Press Connect(d).
    • If you selected the correct port, the Connect button will change to Disconnect.
    • If you selected the incorrect port, the software will pause for a ~5 seconds, and the button will show Retry Connection.
    • Any modules that are powered on before connecting will automatically show up in the GUI.
  5. Power on the wireless modules.
    • The GUI will automatically display the connected module in a new tab.
    • You can manually scan for wireless modules using the Scan For Modules button.(e)

Controlling a Wireless Module

Once connected, the wireless module will show up on the right. Let's get into explaining the GUI:

(a) Tab controls

  • Module Tab Title

    • Click and drag the title to rearrange the tab.
  • Popout

    • Click to pop the tab out into a separate window.
    • You can place the window back into the GUI by dragging it over the GUI.
  • Close

    • Click to close the tab.
    • You can reopen closed module tabs by pressing the Scan For Modules button in the Central tab.

(b) Module Information

  • Module Info

    • The info is formatted as Module [ID number] - v[Version number]
    • Each module has a unique ID.
  • Connection Status
    • The bluetooth icon will change depending on the module's connection status:
      • Connected
      • Disconnected

(c) Module Controls

  • Manual Command

    • Send a manual command to the module. See Manual Module Commands for more information.
  • Sync Module

    • Sync the module with the current GUI settings.
    • For example, if the module has been reset (by power cycle or via reset button), then you can press this button to setup the pins as they appear in the GUI.
  • Reset Module

    • Reset the module.
    • This will stop all functions and turn all of the pins on the module to Custom - Input.
  • Disconnect Module

    • Disconnect the module by turning off its bluetooth connection.
    • Note: to reconnect to this module, you'll have to turn the module off and back on.

(d) Pin Tabs

  • Pin Tabs

    • Switch between the settings for each pin.
    • Each tab can be moved around within the module window. You can have two pin tabs side by side!

(e) Pin Mode Selection

  • Each pin should be set to the mode corresponding to the respective fiber pin function.
  • For example, if Pin 1 on the fiber is an LED pin, then Pin 1 should be set to LED mode here.
  • You can read more about each pin mode in the Pin Modes section. For now, here is a brief overview of each mode:
  • Custom Mode

    • Manually set the input/output of each pin. Useful for testing.
  • Ground Mode

    • Designate the pin as Ground.
    • At least one of the pins should be set to this, which one it is depends on the fiber layout.
  • LED Mode

    • Stimulate using an LED.
  • Temperature Mode

    • Measure temperature using an NTC.

Pin Modes

Custom Mode

Manually set the mode of the internal input/output pins of this pin. Useful for testing fiber functionality.

Use of this mode is NOT recommended unless you know what you're doing.

  • Updates

    • Check the box to receive status updates from the module, such as when the pin is turned on/off.
    • Status updates will appear following Status:.
  • Input Pin Mode

    • Read voltages using this pin.
    • Use Internal Pullup: Check to use the module's internal pullup resistor on this pin.
    • Read Pin: Read the current voltage of the pin. Read value is displayed following Data:.
    • Data Type dropdown: Convert the read voltage between the following:
      • Voltage: The voltage (in V) read on the pin (to 4 decimal places).
      • Raw: The raw value measured by the module (at a resolution of 14 bits.
  • Output Pin Mode

    • Pin on: Check to turn on the pin, uncheck to turn off.

Ground Mode

Use this pin as a Ground pin. Refer to your fiber layout to determine which pin this should be.

LED Mode

Control the fiber LED attached to this pin.

  • Updates

    • Check the box to receive status updates from the module, such as when the pin is pulsing the LED or resting.
    • Status updates will appear following Status:.
  • Manual Control

    • Check the box to manually turn the LED on and off.
  • Pulse Control

    • Control the stimulation waveform of the LED. The chart shows each of the parameters on the waveform.
    • The module will turn the LED on for LED On Duration and off for LED Off Duration repeatedly for the entirety of the Pulse Duration. The module will then turn off the LED and rest for Rest Duration before pulsing again.
    • LED On Duration: The amount of time (in ms) the LED is on before turning off during the pulse period.
    • LED Off Duration: The amount of time (in ms) the LED is off before turning on during the pulse period.
    • Pulse Duration: The amount of time (in ms) the LED on/off cycle should run before resting.
    • Rest Duration: The amount of time (in ms) the module should rest before resuming pulsing.
    • Set All Parameters: Set all of the above parameters at once.
    • Start

      • Start pulsing the LED.
    • Restart

      • Start pulsing the LED from the beginning.
    • Stop

      • Stop pulsing the LED.
  • Test LED Functionality

    • Press Run Test to see if the LED is functioning properly. The result will be following Test Results:.
    • If you see the message LED Malfunction, then the LED in the fiber is being shorted.

Temperature Mode

Read temperature using the NTC attached to this pin.

  • Calibration

    • Input the value of the reference resistor (in Ω) used for this NTC in order to get accurate temperature measurements.
    • Refer to the fiber documentation for this value.
  • Manual Control

    • Read Pin: Read the current voltage of the pin. Read value is displayed following Data:.
    • Data Type dropdown: Convert the read voltage between the following:
      • Temperature: The temperature (in °C) read on the pin (to 3 decimal places).
      • Voltage: The voltage (in V) read on the pin (to 4 decimal places).
      • Raw: The raw value measured by the module (at a resolution of 14 bits.
      • This will also change the data representation on the graph.
  • Automatic Control

    • Continuously read temperature at a certain rate, store it on the module, and send it to the GUI after a certain number of points have been collected.
    • Time Between Data Points: The amount of time (in ms) to wait between reading the current temperature.
      • Note: I do not recommend going below 15ms. Below this, it's possible that the module will not be able to receive any commands (but will still send data).
    • # Points to Collect: The number of points the module should store before sending them all at once to the GUI (max: 1000).
      • Note: Data points are bundled in groups of 100 and then sent to the GUI. When sending 300+ data points (or more), it's possible that a group will get dropped. I advise sending 200 or fewer points at once, if possible.
    • Note: I recommend that you avoid sending faster than every 100ms. This can cause the module to become nonresponsive. If you want to measure every 25ms, for example, try collecting 4 points at once for a send rate of 100ms and still get fast data.
    • Set All Parameters: Set both of the above parameters at once.
    • Start

      • Start reading data.
    • Restart

      • Clear data stored on the module and start reading again.
    • Stop

      • Stop reading data.
  • Graph

    • The graph will plot the data as it is received from the module. The graph can do a few things:
      • Pan the graph using the mouse. Click on the [A] in the bottom left corner to reset the view.
      • Right click on the graph for more options, such as:
        • Locking the X/Y axes to a specific range
        • Exporting the current data on the graph as an image or .csv file
        • And more!
      • Increase the size of the window/tab to increase the size of the graph.
    • The maximum number of stored points is 1,000,000. Once 1,000,000 points have been stored, the GUI will begin to throw out old data.
    • Points to Display: The number of points (backwards in time from now) to display on the graph at once (max 1,000,000).
      • The more points being displayed, the slower the GUI may run. If you are seeing performance issues, try reducing the number of points.
      • The graph will only export what's visible, so even though you may have, say, 50,000 points measured, if Points to Display is 1000, only 1000 points will be exported.
    • Smoothing: Smooth the data using a moving average.
    • Clear Data: Delete all of the data stored in the GUI.

Troubleshooting

  • Can't connect to Central board
    • Make sure that the central board is plugged into your computer.
    • Try selecting a different port from the dropdown. If the port isn't present in the dropdown, try pressing Refresh.
    • If neither of those worked, try:
      1. Close the GUI
      2. Unplug the central board from your computer
      3. Wait 3+ seconds
      4. Plug the central board back into your computer
      5. Open the GUI
      6. Attempt to connect to the central board again
  • Wireless Module keeps disconnecting then reconnecting
    • This is normal due to the nature of Bluetooth. However, if the module fails to reconnect:
      1. Try pressing the Scan for Modules button
      2. If it fails to reconnect, try turning the corresponding wireless module on and off again
    • If neither of these work, see the next issue
  • Wireless Module is nonresponsive
    • This could happen if you are sending data too fast from the module (e.g. reading the temperature every 5ms and sending one point immediately). In this case, you will have to restart the module.
    • If you are sending data at a reasonable rate (e.g. >100ms per batch), then this may be due to the central board crashing or getting frozen. You can try the following steps to remedy the problem:
      1. Disconnect from the central board in the GUI
      2. Unplug the central board from your computer
      3. Wait 3+ seconds
      4. Plug the central board back into your computer
      5. Reconnect to the central board in the GUI
  • LED on the fiber is not lighting up
    • Check that the fiber is plugged into the correct orientation
    • Check that the ground pin is set to the proper pin in the GUI
  • Temperature is displaying incorrect/unexpected values
    • Check that the fiber is plugged into the correct orientation
    • Check that the ground pin is set to the proper pin in the GUI
    • Check that the calibration value is set to the correct resistance

Manual Module Commands

Module commands are groups of arguments separated by commas, with each command ending in a semicolon. E.g. M,N;S,1,P,G; is two commands, M,N; and S,1,P,G; combined. You can combine multiple commands together to send them at once.

The first argument is the command type, and any following arguments will depend on the command's options.

Module Commands - M

These are commands to get general information about the module and perform whole-module tasks

  • Get ID - I
    • Gets the ID of the module
    • Arguments: None
    • Returns: "I[Module ID]"
    • Example usage: M,I;
  • Get Version - V
    • Gets the version of the module
    • Arguments: None
    • Returns: "V[Module Version]"
    • Example usage: M,V;
  • Get Name - V
    • Gets the name of the module
    • Arguments: None
    • Returns: "NModule [Module Version] #[Module ID]"
    • Example usage: M,N;
  • Reset - R
    • Resets the module
    • Arguments: None
    • Returns: "RReset"
    • Example usage: M,R;
  • Disconnect - D
    • Turns of bluetooth on the wireless module
    • Arguments: None
    • Returns: None
    • Example usage: M,D;

Setup Commands - S

These commands set up the pins

Arguments: [Pin Number (0 indexed)], [Pin Option], [Pin Mode]

Pin Options

  • Pin Preset Modes - 'P'
    • Setup the pin according to a preset
    • Arguments: [Pin Mode]
      • 'G': Ground Mode
      • 'L': LED Mode
      • 'N': NTC/Temperature Mode
      • 'D': Data Pin Mode
    • Returns: None
    • Example usage: S,1,P,G;
  • Pin Input Mode - 'I'
    • Set the input pin of the corresponding pin
    • Arguments: [Pin Mode]
      • 'L': Output Low
      • 'H': Output High
      • 'I': Input
      • 'P': Input Pullup
    • Returns: None
    • Example usage: S,1,I,I;
  • Pin Output Mode - 'O'
    • Set the output pin of the corresponding pin
    • Arguments: [Pin Mode]
      • 'L': Output Low
      • 'H': Output High
      • 'I': Input
      • 'P': Input Pullup
    • Returns: None
    • Example usage: S,1,O,I;

Pin Control - P

Control the input/output of each pin individually

Arguments: [Pin Number (0 indexed)], [Pin Option = 'I' or 'O'], [Pin Command]

Pin Commands

  • Read Pin - 'R'
    • Reads the given pin and sends back the data
    • Arguments: None
    • Returns: The read value of the pin
    • Example usage: P,0,I,R;
  • Set Pin Low - 'L' or 'l' or '0'
    • Sets the output to the given pin Low
    • Arguments: None
    • Returns: None
    • Example usage: P,0,O,L;
  • Set Pin High - 'H' or 'h' or '1'
    • Sets the output to the given pin High
    • Arguments: None
    • Returns: None
    • Example usage: P,0,O,H;
  • Receive Updates - U
    • Sets the pin to send updates on state change
    • Arguments: None
    • Returns: A status update when the pin changes
    • Example usage: P,2,I,U;
  • Don't Receive Updates - u
    • Sets the pin to not send updates on state change
    • Arguments: None
    • Returns: None
    • Example usage: P,2,I,u;

LED Control - L

Control the LED pins

Arguments: [LED Pin Number (0 indexed)], [LED Command], [Arguments]

  • LED Setup - C
    • Setup an LED
    • Arguments: [Pulse On Duration (ms)], [Pule Off Duration (ms)], [LED On Duration (ms)], [LED Off Duration (ms)]
    • Returns: None
    • Example usage: L,2,C,1000,4000,10,40;
  • Set Pulse On Duration - N
    • Set the LED Pulse On Duration
    • Arguments: [Pulse On Duration (ms)]
    • Returns: None
    • Example usage: L,2,N,1000;
  • Set Pulse Off Duration - F
    • Set the LED Pulse Off Duration
    • Arguments: [Pulse Off Duration (ms)]
    • Returns: None
    • Example usage: L,2,F,1000;
  • Set LED On Duration - n
    • Set the LED On Duration
    • Arguments: [LED On Duration (ms)]
    • Returns: None
    • Example usage: L,2,n,10;
  • Set LED Off Duration - f
    • Set the LED Off Duration
    • Arguments: [LED Off Duration (ms)]
    • Returns: None
    • Example usage: L,2,f,40;
  • Start LED Pulse - S
    • Starts pulsing the LED
    • Arguments: None
    • Returns: None
    • Example usage: L,2,S;
  • Restart LED Pulse - R
    • Stops then starts pulsing the LED
    • Arguments: None
    • Returns: None
    • Example usage: L,2,R;
  • Stop LED Pulse - s
    • Stops pulsing the LED
    • Arguments: None
    • Returns: None
    • Example usage: L,2,s;
  • Turn LED On - O
    • Turns the LED On
    • Arguments: None
    • Returns: None
    • Example usage: L,2,O;
  • Turn LED Off - o
    • Turns the LED Off
    • Arguments: None
    • Returns: None
    • Example usage: L,2,O;
  • Receive Updates - U
    • Sets the LED to send updates on state change
    • Arguments: None
    • Returns: A status update when the pin changes
    • Example usage: L,2,U;
  • Don't Receive Updates - u
    • Sets the LED to not send updates on state change
    • Arguments: None
    • Returns: None
    • Example usage: L,2,u;

Data Pin Control - D

Control data pins. NTC/Temperature pins are data pins

Arguments: [Data Pin Number (0 indexed)], [Data Pin Command], [Arguments]

  • Setup Data Pin - C
    • Set up a data pin
    • Arguments: [Time Between Data Points (ms)], [Number of Points to Collect]
    • Returns: None
    • Example usage: D,3,C,250,4;
  • Set Time Between Data Points - P
    • Set the time between data points
    • Arguments: [Time Between Data Points (ms)]
    • Returns: None
    • Example usage: D,3,P,250;
  • Set Number of Points to Collect - d
    • Set the number of data points to collect before sending
    • Arguments: [Number of Points to Collect]
    • Returns: None
    • Example usage: D,3,P,4;
  • Read and store - r
    • Read data and store it on the module
    • Arguments: None
    • Returns: Stored data if # points to collect is reached
    • Example usage: D,3,r;
  • Read and send - R
    • Read and send one piece of data
    • Arguments: None
    • Returns: One piece of data
    • Example usage: D,3,R;
  • Flush Data - F
    • Send the number of requested data points from module, and clear stored data
    • Arguments: [Number of points to flush]
    • Returns: One piece of data
    • Example usage: D,3,F,10;
  • Clear Data - c
    • Clear out the stored data on the module
    • Arguments: None
    • Returns: None
    • Example usage: D,3,c;
  • Start Reading Data - S
    • Starts reading data
    • Arguments: None
    • Returns: None
    • Example usage: D,3,S;
  • Restart Reading Data - X
    • Clears stored data on the module then starts reading
    • Arguments: None
    • Returns: None
    • Example usage: D,3,X;
  • Stop Reading Data - s
    • Stops reading data
    • Arguments: None
    • Returns: None
    • Example usage: D,3,s;
  • Receive Updates - U
    • Sets the data pin to send updates on state change
    • Arguments: None
    • Returns: A status update when the pin changes
    • Example usage: D,3,U;
  • Don't Receive Updates - u
    • Sets the data pin to not send updates on state change
    • Arguments: None
    • Returns: None
    • Example usage: D,3,u;
  • Toggle Output High On Read - O
    • The data pin will turn the output on, read the pin, turn it off
    • NTC/Temperature pins enable this by default
    • Arguments: None
    • Returns: None
    • Example usage: D,3,O;
  • Don't Toggle Output High On Read - o
    • The data pin will just read the pin
    • Arguments: None
    • Returns: None
    • Example usage: D,3,o;
  • Set Output High - H
    • Turn on the output side of the data pin
    • Arguments: None
    • Returns: None
    • Example usage: D,3,H;
  • Set Output Low - L
    • Turn off the output side of the data pin
    • Arguments: None
    • Returns: None
    • Example usage: D,3,L;
